# QA Portfolio

Портфолио состоит из двух частей:
- **Автотесты** — что я умею автоматизировать (pytest: API, UI, SQL, тест-дизайн).
- **Баг-репорты** — реальные дефекты, которые я находил в своих проектах.

## Автотесты

Набор автотестов на Python (**pytest**): API, UI, SQL и unit-уровень.
Все тесты запускаемые и проходят.

## Состав

| Файл | Область | Что показывает |
|------|---------|----------------|
| `test_api.py` | API-тестирование (`requests`) | Статус-коды (200/201/404), схема и типы JSON, негативные кейсы, параметризация |
| `test_ui_playwright.py` | UI / E2E (Playwright) | Автоматизация браузера: переход, ввод, клик, проверка; скриншот при падении; точные локаторы |
| `test_sql.py` | SQL / БД (SQLite) | SELECT/WHERE/JOIN/GROUP BY/агрегаты + проверки целостности (нет отрицательных балансов, нет «сирот», ограничения CHECK) |
| `test_subtitle_qa.py` | Тест-дизайн | Критерии приёмки для недетерминированного ML-вывода; граничный анализ |
| `subtitle_qa.py` | — | Модуль под тестом |

## Запуск

```bash
pip install pytest pytest-playwright requests
playwright install chromium
pytest
```

> Автотесты гоняются на нейтральных учебных целях (JSONPlaceholder, TodoMVC),
> чтобы их можно было склонировать и запустить где угодно. Это демонстрация
> навыка автоматизации; модуль `subtitle_qa` вытащен из реального проекта.

## Баг-репорты

[`Баг-репорты.pdf`](./Баг-репорты.pdf) — **19 реальных дефектов** из моих проектов
в стандартном формате: Шаги · Ожидаемо · Фактически · Severity/Priority · Причина ·
Фикс · Проверка. Найдены ручным и исследовательским тестированием + анализом логов.

Категории: функциональные, UI-автоматизация, гонки/параллелизм, производительность,
безопасность (path traversal, RCE, инъекции), целостность данных, интеграции/сеть,
локализация.
